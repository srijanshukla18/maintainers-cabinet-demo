# Contributing to cabinet-demo

Thanks for your interest in contributing!

## Before you open an issue

- Search existing issues to avoid duplicates.
- For bugs, include: version, environment, reproduction steps, expected behavior, actual behavior.

## Before you open a PR

- Open an issue first for non-trivial changes.
- Add tests for any changed behavior in `src/`.
- Update docs if you change CLI flags or public API.
- Run `npm test` before submitting.

## Commit style

```
type: short description

- bullet points for detail
```

Types: `feat`, `fix`, `docs`, `test`, `refactor`, `ci`
